## Проблемы текущей реализации

### 1. Формат диалога
**Проблема:** Текущий диалог — это структурированная форма с фиксированными вопросами по порядку, а не свободный чат-бот.

**Текущая реализация:**
- Фиксированная последовательность вопросов (`FIELD_SEQUENCE`)
- Пользователь не может задавать вопросы в произвольном порядке
- Нет возможности уточнить детали после ответа
- Нет контекстного понимания — агент просто собирает ответы

**Что нужно (по заданию):**
- Свободный диалог, где пользователь может задавать вопросы
- Агент должен понимать контекст и уточнять детали
- Возможность вернуться к предыдущим темам

### 2. Отсутствие метрик
**Проблема:** Нет измерения ключевых метрик успеха из задания.

**Что нужно добавить:**
- Логирование времени генерации документов
- Отслеживание успешности завершения диалогов
- Сбор статистики использования

### 3. Лидирующие индикаторы
**Проблема:** В задании упомянуты отдельно, но в проекте нет отдельной генерации.

**Что нужно:** Отдельный генератор или раздел для лидирующих индикаторов (leading indicators).

### 4. Интеграция с Confluence
**Проблема:** Дополнительная функция не реализована (хотя это опционально).

4. **Улучшить UX диалога:**

# Business Requirements Document (BRD)

## 1. Обзор проекта

**Название:** Платформа онлайн-обучения для корпоративных клиентов

**Цель:** Создать платформу онлайн-обучения для корпоративных клиентов с видеолекциями, тестами и отслеживанием прогресса

Платформа онлайн-обучения представляет собой комплексное решение для корпоративного обучения, которое позволяет компаниям эффективно обучать своих сотрудников через веб-платформу и мобильное приложение. Система обеспечивает доступ к видеолекциям, интерактивным тестам, сертификатам и детальной отчетности по прогрессу обучения.

### 1.1 Описание проекта

Платформа онлайн-обучения для корпоративных клиентов представляет собой веб-платформу и мобильное приложение, обеспечивающее доступ к курсам, просмотру видеолекций, прохождению тестов, получению сертификатов и отслеживанию прогресса обучения. Администраторы могут создавать курсы и назначать их сотрудникам, что позволяет компаниям централизованно управлять процессом обучения и развитием навыков персонала.

Основная проблема, которую решает платформа: сотрудники компаний не имеют доступа к качественному корпоративному обучению, обучение в офлайн-формате дорого и неудобно, отсутствует централизованная система учета навыков.

## 2. Цели и KPI

### 2.1 Ключевые цели

1. Регистрация 1000 активных пользователей в первый квартал после запуска платформы
2. Достижение показателя завершения курсов на уровне 70% от назначенных курсов
3. Получение средней оценки платформы 4.5 из 5 баллов от пользователей
4. Привлечение 20 корпоративных клиентов в первый год работы платформы

### 2.2 KPI

| Критерий | Описание |
|----------|----------|
| Количество активных пользователей | 1000 зарегистрированных активных пользователей в первый квартал |
| Процент завершения курсов | 70% пользователей завершают назначенные курсы |
| Средняя оценка платформы | 4.5 балла из 5 по оценкам пользователей |
| Количество корпоративных клиентов | 20 корпоративных клиентов в первый год |

## 3. Стейкхолдеры и роли

### 3.1 Стейкхолдеры

| Стейкхолдер | Описание |
|-------------|----------|
| HR-отделы компаний-клиентов | Заинтересованные стороны, которые используют платформу для обучения сотрудников |
| Учащиеся-сотрудники | Основные пользователи платформы, которые проходят обучение |
| Администраторы курсов | Создают и управляют контентом курсов |
| Менеджеры по обучению | Координируют процесс обучения и анализируют результаты |
| IT-отдел разработки | Отвечает за техническую реализацию и поддержку платформы |
| Отдел продаж | Привлекает новых корпоративных клиентов |

### 3.2 Роли

| Роль | Описание |
|------|----------|
| Учащийся | Просматривает курсы, проходит тесты, отслеживает свой прогресс обучения |
| Инструктор | Создает контент курсов, проверяет задания, обеспечивает обратную связь учащимся |
| Администратор HR | Назначает курсы сотрудникам, формирует отчеты по обучению, управляет доступом |
| Системный администратор | Управляет пользователями, настраивает систему, обеспечивает техническую поддержку |

## 4. Требования и бизнес-правила

### 4.1 Функциональные требования

1. Разработка веб-платформы и мобильного приложения для доступа к курсам
2. Реализация системы управления курсами с возможностью создания и редактирования контента
3. Интеграция видеоплеера с поддержкой субтитров для просмотра лекций
4. Разработка модуля тестирования с различными типами вопросов
5. Реализация системы отчетности по прогрессу обучения с детальной аналитикой
6. Интеграция с корпоративными системами HR для синхронизации данных о сотрудниках

### 4.2 Бизнес-правила

| Требование | Описание |
|------------|----------|
| Доступ к курсам | Доступ к курсу предоставляется только назначенным сотрудникам |
| Условия выдачи сертификата | Сертификат выдается при успешном прохождении курса при условии не менее 80% правильных ответов в тестах |
| Срок прохождения курса | Курсы можно проходить в любое время в течение 90 дней с момента назначения |
| Сохранение прогресса | Прогресс обучения сохраняется при перерыве, пользователь может продолжить с того места, где остановился |

## 5. Ограничения и допущения

### 5.1 Ограничения

1. Бюджет проекта составляет 5 млн тенге, что определяет масштаб и функциональность MVP
2. Срок разработки ограничен 8 месяцами, что требует четкого планирования и приоритизации функций
3. Поддержка только русского и казахского языков на этапе MVP, другие языки будут добавлены в последующих версиях
4. Интеграция только с основными HR-системами, расширенная интеграция планируется в будущем
5. Платформа должна работать на мобильных устройствах с iOS 13+ и Android 8+, более старые версии не поддерживаются

### 5.2 Допущения

1. Клиенты будут предоставлять качественный контент для курсов, готовый к размещению на платформе
2. Интернет-соединение у большинства пользователей стабильное и достаточное для просмотра видеолекций
3. Компании готовы инвестировать в обучение сотрудников и использовать платформу на регулярной основе
4. Пользователи имеют базовые навыки работы с мобильными приложениями и веб-интерфейсами

## 6. Риски

| Риск | Описание |
|------|----------|
| Низкая вовлеченность пользователей | Риск низкой вовлеченности пользователей в обучение может привести к невыполнению KPI по завершению курсов |
| Технические проблемы с загрузкой видео | Проблемы с загрузкой видео на медленном интернете могут негативно повлиять на пользовательский опыт |
| Конкуренция с существующими платформами | Наличие конкурентов с устоявшимися решениями может затруднить привлечение клиентов |
| Сложность интеграции с корпоративными системами | Технические сложности при интеграции с различными HR-системами могут привести к задержкам в разработке |
| Риски по защите персональных данных | Необходимость обеспечения соответствия требованиям по защите персональных данных требует дополнительных мер безопасности |

## 7. Нефункциональные требования

### 7.1 Производительность

1. Время загрузки страницы должно быть менее 3 секунд при стандартном интернет-соединении
2. Система должна поддерживать до 5000 одновременных пользователей без деградации производительности
3. Видеоплеер должен поддерживать стриминг в разрешении 720p и 1080p с адаптивным битрейтом
4. Поддержка офлайн-режима для мобильного приложения с синхронизацией данных при восстановлении соединения

### 7.2 Безопасность

1. Данные должны храниться в соответствии с требованиями GDPR и местного законодательства о защите персональных данных
2. Доступность платформы должна составлять не менее 99.5% времени работы в течение месяца
3. Все персональные данные должны быть зашифрованы при передаче и хранении
4. Система должна обеспечивать безопасную аутентификацию и авторизацию пользователей

### 7.3 Масштабируемость

1. Архитектура платформы должна позволять горизонтальное масштабирование для увеличения нагрузки
2. База данных должна быть оптимизирована для быстрого доступа к контенту курсов и данным пользователей
3. Система должна поддерживать добавление новых функций без значительных изменений в существующей архитектуре

## 8. Описание процесса

HR-администратор создает курс в системе и назначает его группе сотрудников. Сотрудники получают уведомление о новом назначенном курсе через мобильное приложение или веб-платформу. Пользователь открывает мобильное приложение, просматривает видеолекции в удобное для себя время, проходит тесты после каждого модуля для проверки усвоения материала. Система автоматически отслеживает прогресс обучения каждого пользователя, фиксируя просмотренные лекции и результаты тестирования. При успешном завершении курса система автоматически выдает сертификат пользователю. Администратор имеет доступ к детальным отчетам о прохождении курсов, может видеть статистику по каждому сотруднику и назначать дополнительные курсы на основе результатов обучения.
